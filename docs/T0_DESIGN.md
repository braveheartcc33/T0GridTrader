## 2. 核心逻辑（破网停机版）

### 2.1 设计原则

本版本采用**"破网停机，尾盘回归"**策略：

> **价格超出网格边界（±5档）→ 停止交易 → 14:30 尾盘统一平仓**

这是比"连续N格"更稳健的方案，物理意义明确，代码简单，不易出错。

### 2.2 网格边界熔断机制

盘中 `check_and_trade` 每4秒执行一次：

1. 计算当前档位 `current_level = round((price - base_price) / spacing)`
2. 如果 `current_level > MAX_LEVEL`（>+5）→ **暂停交易，等待尾盘**
3. 如果 `current_level < MIN_LEVEL`（<-5）→ **暂停交易，等待尾盘**
4. 如果档位在边界内 → 执行正常网格买卖

### 2.3 为什么"破网停机"更安全？

- 超出边界意味着价格趋势已经非常强（持续上涨或下跌）
- 继续追卖/追买可能会买在高位/卖在低位
- 停止交易，等待尾盘平仓，保证持仓回归昨日水平
- 避免了"连续N格"方案在震荡市中的频繁误判问题

## 3. T+0 规则

### 3.1 规则本质

A 股 T+0 约束的是**当日净卖出量**：
`当日累计卖出量 - 当日累计买入量 ≤ 底仓数量（10,000 股）`

### 3.2 关键公式

> 注意：A股T+0当天买的不能卖，所以买入不增加可卖出额度！

```
net_short = cumulative_sells  # 只算卖出！累计净卖出
available_to_sell = base_position - net_short
```

### 3.3 T+0 交易示例

| 操作 | 累计卖 | 累计买 | net_short | 可卖出数量 |
|------|-------|-------|-----------|-----------|
| 开盘 | 0 | 0 | 0 | 10,000 |
| 卖 3,000 股 | 3,000 | 0 | 3,000 | 7,000 |
| 买 2,000 股 | 3,000 | 2,000 | 3,000 | 7,000 |
| 卖 5,000 股 | 8,000 | 2,000 | 8,000 | 2,000 |

> 买 2,000 股后，虽然"net_short"理论上减少到1,000，但买入的2,000股当日不能卖，所以可卖出额度**不增加**，仍为7,000。

## 4. 盘中正常交易逻辑（峰值/谷值版）

> 本版本采用"峰值/谷值"追踪，避免在震荡市中错误追加交易

当价格档位在 [-5, +5] 范围内时：

1. 计算 `price_diff = current_price - prev_price`
2. 如果 `|price_diff| < spacing × 80%` → **噪声过滤，不交易**
3. 如果 `price_diff > 0` → **向上穿越格子边界**
   - **只在新高位时卖出**：必须 `current_level > peak_level` 才卖出
   - 卖出后更新 `peak_level = current_level`
4. 如果 `price_diff < 0` → **向下穿越格子边界**
   - **只在新低位时买回**：必须 `current_level < valley_level` 才买回
   - 买回后更新 `valley_level = current_level`

> 当 `current_position == base_position`（持仓回归底仓）时，重置 `peak_level = 0, valley_level = 0`

### 卖出逻辑
- 计算 `net_short = cumulative_sells`（买的不算！）
- `available_to_sell = base_position - net_short`
- `can_sell = min(穿越格子数 × 每格股数, available_to_sell)`

## 5. 尾盘强制平仓（14:30）

**规则**：无视 T+0 约束，将持仓恢复到**昨日收盘数量**

```
14:30 触发 force_close_all_t0():
  while current_position != yesterday_position:
    if current_position > yesterday_position:
      SELL 差值股
    elif current_position < yesterday_position:
      BUY 差值股
```

## 7. 代码实现要点

### 7.1 GridEngine 核心属性

```python
self.MAX_LEVEL = grid_count // 2   # = 5
self.MIN_LEVEL = -self.MAX_LEVEL   # = -5
self.cumulative_sells = 0         # 累计卖出
self.cumulative_buys = 0           # 累计买回
self.yesterday_position = 10000     # 昨日收盘持仓
```

### 7.2 check_and_trade 流程图

```
输入: current_price
  ↓
止损检查（亏损≥2%则清仓）
  ↓
尾盘检查（14:30后强制平仓）
  ↓
非交易时段检查
  ↓
网格边界检查
  ↓ [current_level > MAX_LEVEL] → 暂停，等待尾盘
  ↓ [current_level < MIN_LEVEL] → 暂停，等待尾盘
  ↓ [边界内]
网格交易（涨卖跌买）
  ↓
输出: (TradeRecord or None, avg_cost)
```

### 7.3 force_close_all_t0

```python
def force_close_all_t0(self, current_price, current_time=None):
    while self.current_position != self.yesterday_position:
        if self.current_position > self.yesterday_position:
            self._record_trade("SELL", ...)  # 多退
        else:
            self._record_trade("BUY", ...)   # 少补
```

## 8. 与旧版（连续5格版）的对比

| 项目 | 旧版（连续5格） | 新版（破网停机） |
|------|----------------|----------------|
| 触发条件 | 连续穿越5格 | 超出±5档 |
| 计数器 | 需维护方向+计数 | 无需计数 |
| 震荡市 | 容易误判 | 无误判 |
| 代码复杂度 | 高（多状态） | 低（边界判断） |
| 安全性 | 中等 | 更高 |

## 9. 注意事项

- **噪声过滤**：80%阈值，防止格子内震荡产生无效交易
- **止损规则**：亏损≥2%时强制清仓，当日不再交易
- **收盘前30分钟**：尾盘窗口，触发强制平仓
- **新交易日**：调用 `reset_day()` 重置所有计数器
