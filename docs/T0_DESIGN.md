### 交易标的与参数

| 参数 | 值 |

| 网格档位 | 11 档（-5 到 +5） |

| 网格边界下限 | -5 档 |

### 网格档位定义

以基准价（今日开盘价）为档位 0：

| 档位 | 价格 |
|------|------|
| -5 | 基准价 × (1 - 5 × ATR/10) |
| ... | ... |
| 0 | 基准价（今日开盘价） |
| ... | ... |
| +5 | 基准价 × (1 + 5 × ATR/10) |

> 实际价格随 ATR 和基准价动态变化，以上为示意

## 2. 核心逻辑

### 2.1 设计原则

本版本采用**"破网停机，尾盘回归"**策略：

> **价格超出网格边界（±5档）→ 停止交易 → 14:30 尾盘统一平仓**

这是比"连续N格"更稳健的方案，物理意义明确，代码简单，不易出错。

### 2.2 网格边界熔断机制

每格间距 = `ATR(14) / 10 × 时间倍数`，根据不同时段自动调整：

| 时段 | 时间倍数 | 间距(ATR=0.0266) |
|------|---------|-----------------|
| 开盘 9:30 | x4.0 | 0.0106 |
| 10:00 | x3.0 | 0.0080 |
| 11:00 | x2.0 | 0.0053 |
| 11:30 | x3.0 | 0.0080 |
| 13:00 | x4.0 | 0.0106 |
| 13:30 | x3.0 | 0.0080 |
| 14:30 尾盘 | x2.0 | 0.0053 |
| 15:00 收盘 | 不交易 | - |

盘中 `check_and_trade` 每4秒执行一次：

1. 计算当前档位 `current_level = round((price - base_price) / spacing)`
2. 如果 `current_level > MAX_LEVEL`（>+5）→ **暂停交易，等待尾盘**
3. 如果 `current_level < MIN_LEVEL`（<-5）→ **暂停交易，等待尾盘**
4. 如果档位在边界内 → 执行正常网格买卖

### 2.3 为什么"破网停机"更安全？

- 超出边界意味着价格趋势已经非常强（持续上涨或下跌）
- 继续追卖/追买可能会买在高位/卖在低位
- 停止交易，等待尾盘平仓，保证持仓回归昨日水平
- 避免了"连续N格"方案在震荡市中的频繁误判问题

## 3. T+0 规则

### 3.1 规则本质

A 股 T+0 约束的是**当日净卖出量**：
`当日累计卖出量 - 当日累计买入量 ≤ 底仓数量（10,000 股）`

### 3.2 关键公式

> 注意：A股T+0当天买的不能卖，所以买入不增加可卖出额度！

```
# 可卖出 = 底仓 - 今日累计卖出（买的不算！买不增加可卖额度）
available_to_sell = max(0, base_position - cumulative_sells)
```

### 3.3 T+0 额度说明

| 操作 | 累计卖 | 累计买 | 可卖出额度 | 可买入额度 |
|------|-------|-------|-----------|-----------|
| 开盘 | 0 | 0 | 10,000 | 10,000 |
| 卖 3,000 股 | 3,000 | 0 | 7,000 | 10,000 |
| 买 2,000 股 | 3,000 | 2,000 | 7,000 | 8,000 |
| 卖 5,000 股 | 8,000 | 2,000 | 2,000 | 8,000 |

> 累计买入上限 = 底仓(10000)，累计卖出上限 = 底仓(10000)。
> 买入不增加可卖出额度（新逻辑下，用目标持仓计算，买的股票已经在差值里考虑了）。

## 4. 盘中正常交易逻辑

> 核心原则：每一格的变化 = 买卖1份（1000股），持仓与目标持仓对齐

### 4.1 档位与目标持仓

| 档位 | 目标持仓 | 与底仓的差 |
|------|---------|-----------|
| +5 | 5000 | -5000（卖5份） |
| +4 | 6000 | -4000（卖4份） |
| +3 | 7000 | -3000（卖3份） |
| +2 | 8000 | -2000（卖2份） |
| +1 | 9000 | -1000（卖1份） |
| **0** | **10000** | **底仓** |
| -1 | 11000 | +1000（买1份） |
| -2 | 12000 | +2000（买2份） |

### 4.2 核心约束（两条铁律）

**铁律1 - T+0额度限制**：
- 累计买入上限 = 底仓(10000)
- 累计卖出上限 = 底仓(10000)
- 额度用完即止，超出部分直接裁剪

**铁律2 - 目标仓位映射**：
- 公式：`目标持仓 = 底仓 - 档位 × 每格股数`
- 交易量：`trade = 当前持仓 - 目标持仓`（正=卖，负=买）
- 实际下单量 = 截断(理论交易量, 剩余额度)

### 4.3 交易示例

假设：基准价=0.717，间距=0.0027（实际=ATR14/11≈0.0024），每格=1000股，底仓=10000股

| 时刻 | 价格 | 档位 | 变化 | 目标持仓 | 当前持仓 | 交易 | 原因 |
|------|------|------|------|---------|---------|------|------|
| T1 | 0.725 | +1 | 0→+1 | 9000 | 10000 | 卖1000 | 持仓>目标 |
| T2 | 0.733 | +2 | +1→+2 | 8000 | 9000 | 卖1000 | 持仓>目标 |
| T3 | 0.725 | +1 | +2→+1 | 9000 | 8000 | 买1000 | 持仓<目标 |
| T4 | 0.717 | 0 | +1→0 | 10000 | 9000 | 买1000 | 持仓<目标 |
| T5 | 0.709 | -1 | 0→-1 | 11000 | 10000 | 买1000 | 持仓<目标 |

> 新间距0.0080（x3.0），每格价格变化更大，信号更少更精准

## 5. 尾盘强制平仓（14:30）

**规则**：无视 T+0 约束，将持仓恢复到**昨日收盘数量**

```
14:30 触发 force_close_all_t0():
  while current_position != yesterday_position:
    if current_position > yesterday_position:
      SELL 差值股
    elif current_position < yesterday_position:
      BUY 差值股
```

## 6. 完整交易流程示例

> 以下为示意，参数随配置的股票动态变化

**示例参数**：基准价=0.717元，ATR(14)=0.0266，每格间距=0.0080（x3.0倍），底仓=10,000股

| 时刻 | 价格 | 档位 | 操作 | 持仓 | 状态 |
|------|------|------|------|------|------|
| 开盘 | 0.717 | 0 | - | 10,000 | 正常 |
| 9:31 | 0.720 | +1 | 卖 1000 | 9,000 | 正常 |
| 9:35 | 0.723 | +2 | 卖 1000 | 8,000 | 正常 |
| 10:05 | 0.720 | +1 | 买 1000 | 9,000 | 正常 |
| 10:15 | 0.717 | 0 | 买 1000 | 10,000 | 正常 |

## 7. 代码实现要点

### 7.1 GridEngine 核心属性

```python
self.MAX_LEVEL = grid_count // 2   # = 5
self.MIN_LEVEL = -self.MAX_LEVEL   # = -5
self.cumulative_sells = 0          # 累计卖出（买的不算）
self.current_level = 0             # 当前档位（用于检测档位变化）
self.yesterday_position = 10000     # 昨日收盘持仓
```

### 7.2 check_and_trade 核心逻辑

```python
def check_and_trade(self, current_price, current_time=None):
    # 1. 止损检查
    # 2. 尾盘强制平仓
    # 3. 非交易时段检查
    # 4. 网格边界熔断

    # 5. 正常网格交易
    # 只有档位变化时才交易
    current_level = self._price_to_level(current_price)
    if current_level == self.current_level:
        return None  # 档位未变，不交易

    # 目标持仓 = 底仓 - 档位 × 每格股数
    target = self.base_position - current_level * self.shares_per_grid
    trade_shares = self.current_position - target

    if trade_shares > 0:
        # 卖出（受T+0约束）
        available = self.base_position - self.cumulative_sells
        actual = min(trade_shares, max(0, available))
        if actual > 0:
            self._record_trade("SELL", current_price, actual, current_level, ...)
            self.current_level = current_level
    else:
        # 买入
        self._record_trade("BUY", current_price, -trade_shares, current_level, ...)
        self.current_level = current_level
```

## 8. 与旧版对比

## 9. 注意事项

- **噪声过滤**：80%阈值，防止格子内震荡产生无效交易
- **止损规则**：亏损≥2%时强制清仓，当日不再交易
- **收盘前30分钟**：尾盘窗口，触发强制平仓
- **新交易日**：调用 `reset_day()` 重置所有计数器
